Camadas de rede: o mapa completo, com foco em L4 e L7

A Lição 05 jogou dois rótulos sem explicar: ALB = camada 7 e NLB = camada 4. Esta bônus mostra as 7 camadas inteiras — mas o destino é claro: entender por que essas duas decidem a escolha do load balancer.

🎯 Por que esta bônus? "Camada 4/7" é jargão de rede que trava muita gente de software. Ter o mapa todo na cabeça — e saber que só duas camadas importam para o balanceador — te dá vocabulário para conversar com qualquer pessoa de infra de igual pra igual, exatamente a autoridade técnica que a sua missão busca. ⏱️ ~9 min · quiz de 5 questões. Pré-requisito: Lição 05 (ELB).
O que cai na prova: o CLF-C02 cobra de verdade só camada 4 (NLB) e camada 7 (ALB). As outras 5 camadas aqui são fluência fundamental para sua missão de liderança — aprenda o mapa, mas não sue para decorar a lista.

A ideia: camadas de abstração que se embrulham

A rede é organizada em camadas, igual ao software: cada uma esconde a de baixo. O modelo de referência é o OSI, com 7 camadas. Sua mensagem desce a pilha no remetente — cada camada adiciona seu próprio embrulho (cabeçalho) — viaja como sinal físico, e sobe a pilha no destino, sendo desembrulhada camada a camada.1

⬇️ remetente: aplicação → … → bits no fio  ·  ⬆️ destino: bits → … → aplicação

As 7 camadas, cada uma como uma pergunta

Não decore a ordem — siga a lógica: cada camada responde uma pergunta necessária para a mensagem viajar e ser entendida. As duas destacadas (load balancer) são as que você vai usar de verdade.

↑ Camadas de "host" — onde vive o software
7
Aplicação ALB
"Qual protocolo da aplicação e o que o usuário quer?"
HTTP/HTTPS, DNS, SMTP, SSH · AWS: ALB, API Gateway, CloudFront, Route 53
6
Apresentação
"Em que formato? Cifrado? Comprimido?"
TLS/SSL, codificação (UTF-8), JPEG · AWS: certificados via ACM (HTTPS)
5
Sessão
"Quem abre, mantém e fecha a conversa?"
Controle de diálogo (na prática, quase sempre fundida no app)
4
Transporte NLB
"Entrega confiável ou rápida? Para qual porta/processo?"
TCP (confiável) vs UDP (rápido) + portas · AWS: NLB, Security Groups, NACLs
↓ Camadas de "mídia" — como os dados realmente trafegam
3
Rede
"Por qual rota chego ao destino, entre redes diferentes?"
IP, roteamento, routers · AWS: VPC, sub-redes, route tables
2
Enlace (Data Link)
"Como entrego ao próximo salto no mesmo segmento?"
MAC, switches, Ethernet, Wi-Fi, ARP · AWS: abstraído
1
Física
"Como bits viram sinais no mundo real?"
Cabos, fibra, rádio, voltagem · AWS: hardware do data center (você nunca toca)
🧠 O presente que a nuvem te dá: on-premises você cuidaria das camadas 1 e 2 (cabos, switches). Na AWS, 1 e 2 somem para você e você opera da camada 3 (VPC) para cima. É a vantagem "parar de manter data centers" vista pela lente das camadas.

O foco: camada 4 vs camada 7 (NLB vs ALB)

De todas as 7, só duas decidem o tipo de load balancer — porque a pergunta é uma só: quão fundo o balanceador abre o pacote?

📨 Analogia do envelope. A camada 4 é o endereço escrito no envelope (IP + porta): dá para entregar sem abrir. A camada 7 é abrir o envelope e ler a carta (o caminho da URL, o cabeçalho Host, o cookie) para então decidir a rota.
NLB — camada 4ALB — camada 7
O que enxergaIP + porta (não abre o envelope)Conteúdo HTTP: URL, host, headers, cookies
Pode rotear por…destino TCP/UDPcaminho (/api → grupo X), host, método…
Velocidade⚡ extrema — milhões de req/s, latência mínimaum pouco mais de overhead (ele lê e interpreta)
ProtocoloTCP / UDP (qualquer coisa)HTTP / HTTPS (web)
Use quando…performance/latência extrema, TCP não-HTTP, IP estáticoapp web, roteamento por conteúdo, redirecionar/autenticar
A regra de decisão em uma frase: precisa que o balanceador leia o HTTP para decidir (rotear /api e /site para grupos diferentes)? → ALB (L7). Só precisa de velocidade bruta encaminhando pacotes? → NLB (L4). Mais inteligência custa um pouco de overhead; menos inteligência rende performance máxima.
Cuidado com a intuição "L7 é melhor que L4": não é melhor, é diferente. O NLB é superior justamente onde você não quer que ninguém abra o envelope — latência mínima e throughput gigante. Quem decide é o trabalho, não o número da camada. (Mesma lógica do RI vs Savings Plans: não existe "o melhor", existe "o certo para a carga".)

OSI vs. o modelo real (TCP/IP)

O OSI é didático (7 camadas). A internet roda o TCP/IP, mais enxuto (~4 camadas). "Camada 4" e "camada 7" sobrevivem no dia a dia porque existem nos dois modelos e batem com o que os balanceadores fazem.2

Conectando à sua missão

Quando alguém perguntar "por que ALB e não NLB?", a resposta defensável não é "porque é o padrão" — é: "nosso tráfego é HTTP e precisamos rotear por caminho de URL para microsserviços diferentes; isso exige um balanceador de camada 7". E numa investigação de incidente, a camada vira um localizador: "o ping responde (L3 ok) mas a porta recusa (L4)". Falar nessa régua é decidir e diagnosticar de igual pra igual com infra.
